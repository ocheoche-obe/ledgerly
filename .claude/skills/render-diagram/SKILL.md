# Render an architecture diagram (diagram as code)

> PORTABLE SKILL — project-agnostic. Copy this folder into any project's
> `.claude/skills/`. Nothing below assumes a specific project or cloud, though the icon
> imports shown are AWS (the `diagrams` library also ships Azure, GCP, k8s, and on-prem
> icon sets).

## The convention this skill enforces

**Diagrams are code.** The architecture diagram is generated by a Python script that
lives in `docs/` next to the architecture doc, and its PNG/PDF outputs are committed
alongside it. Whenever the system shape changes, update the script and re-render **in the
same commit** — the picture must never drift from the doc. Hand-drawn or GUI-tool
diagrams are not used.

## 1. Setup (once per machine)

```bash
pip install diagrams        # or: python3 -m venv .venv && .venv/bin/pip install diagrams
brew install graphviz       # provides `dot`; Linux: apt install graphviz
```

If the system Python is externally managed (Homebrew/Debian), prefer the venv form.

## 2. Script conventions (the load-bearing part)

Follow AWS official architecture-diagram style:

1. **Left-to-right flow** — `direction="LR"`. The user/actor enters on the left; the
   request descends through frontend → auth → API → compute → data toward the right.
   A second flow (async pipeline, batch) reads left-to-right as its own horizontal band.
2. **Sibling layer clusters, not deep nesting** — one outer cloud-boundary cluster, then
   flat named clusters per layer ("Frontend", "API Layer", "Data", …) so graphviz lays
   out a clean grid.
3. **Orthogonal edges** — `splines="ortho"` in `graph_attr` gives straight, right-angled
   lines (curvy default splines overlap and cross; that's the classic complaint).
4. ⚠ **THE PITFALL: with `splines="ortho"`, regular edge `label`s are misplaced** —
   they float away from their edge or collide with cluster titles. Use graphviz
   **external labels** instead: `Edge(**{"xlabel": "presigned PUT"})`, never
   `Edge(label=...)`. Keep labels short (1–3 words).
5. **`newrank: "true"`** in `graph_attr` — global ranking across clusters; much cleaner
   grids.
6. **Declaration order shapes band order** — declare clusters in the vertical order you
   want the bands to appear (interactive path first, async pipeline second, ops last).
7. **Invisible edges are the layout pins** — `a - Edge(style="invis") - b` nudges
   graphviz to place things in the same band/rank without drawing a line. Use them to
   keep an ops cluster beside the data layer, or to hold two bands apart.
8. **Edge semantics** — solid arrows = data/request flow; `style="dotted"` = control
   plane, auth verification, and failure/retry paths.
9. **Output next to the script** — derive the filename from the script location so it
   works from any cwd, and emit both formats:
   `filename=str(Path(__file__).resolve().parent / "<project>-architecture-diagram")`,
   `outformat=["png", "pdf"]`.
10. **Node captions carry the detail** — put specifics (service config, key constraints,
    "IAM auth — no API key") in `\n`-separated node labels, not in edge labels.

### Starter template

```python
from pathlib import Path
from diagrams import Diagram, Cluster, Edge
from diagrams.aws.compute import Lambda            # swap icon sets per provider:
from diagrams.aws.database import Dynamodb         # diagrams.azure.*, diagrams.gcp.*,
from diagrams.aws.network import CloudFront, APIGateway
from diagrams.aws.storage import S3
from diagrams.onprem.client import User

graph_attr = {
    "fontsize": "18", "labelloc": "t", "pad": "0.5",
    "nodesep": "0.9", "ranksep": "1.3",
    "splines": "ortho",      # straight right-angle edges (AWS style)
    "newrank": "true",       # global ranking across clusters
    "bgcolor": "white",
}
CLOUD_ATTR = {  # provider boundary: tinted fill, dashed border
    "bgcolor": "#F2F8FE", "pencolor": "#147EBA", "fontcolor": "#147EBA",
    "style": "dashed,rounded", "penwidth": "2", "fontsize": "14",
    "labeljust": "l", "margin": "22",
}
LAYER_ATTR = {  # per-layer cluster: white fill, thin grey border
    "bgcolor": "#FFFFFF", "pencolor": "#888888", "fontcolor": "#444444",
    "style": "rounded", "penwidth": "1", "fontsize": "11",
    "labeljust": "l", "margin": "14",
}

OUT = str(Path(__file__).resolve().parent / "PROJECT-architecture-diagram")

with Diagram("PROJECT — System Architecture", show=False, direction="LR",
             filename=OUT, outformat=["png", "pdf"],
             graph_attr=graph_attr, node_attr={"fontsize": "11"},
             edge_attr={"fontsize": "10"}):
    user = User("User")
    with Cluster("Cloud boundary", graph_attr=CLOUD_ATTR):
        with Cluster("Frontend", graph_attr=LAYER_ATTR):
            cdn = CloudFront("CDN")
        with Cluster("API Layer", graph_attr=LAYER_ATTR):
            api = APIGateway("API")
        with Cluster("Compute", graph_attr=LAYER_ATTR):
            fn = Lambda("handler")
        with Cluster("Data", graph_attr=LAYER_ATTR):
            db = Dynamodb("table")

    user >> cdn
    user >> Edge(**{"xlabel": "JWT"}) >> api       # xlabel, never label (see pitfall)
    api >> fn >> db
    # layout pin example: keep two bands aligned without drawing a line
    # cdn - Edge(style="invis") - api
```

## 3. Render and — mandatory — inspect

```bash
python docs/render_architecture.py
```

Then **open/Read the PNG and actually look at it**. Never ship the first render unseen.
Check for, and fix by iterating:

- labels colliding with cluster titles or the diagram title (→ shorten, or confirm
  they're `xlabel`s)
- labels floating far from their edge (→ the `label` vs `xlabel` pitfall)
- bands in the wrong vertical order (→ reorder cluster declarations, add invisible pins)
- long edges cutting through unrelated clusters (→ add invisible pins, raise
  `ranksep`/`nodesep`)

Two or three render→inspect iterations are normal.

## 4. Wire into the docs

- The architecture doc's diagram section links the PNG + PDF and the script, embeds the
  PNG inline, and states the rule: *any change to the system shape updates the script and
  re-renders in the same commit*.
- Keeping a small ASCII sketch in the doc as a text-searchable quick reference is fine —
  mark the rendered diagram as authoritative if they disagree.
- Commit script + PNG + PDF together.
